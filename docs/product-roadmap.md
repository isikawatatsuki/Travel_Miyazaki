# 旅のしおり 機能棚卸し

## 現在の基盤

- 旅の基本設定：旅行名、日程、出発・到着時刻、交通、宿泊先
- 日別予定：時刻未定を含む追加・編集・削除、地図リンク
- 地図：出発地から宿泊先までの経路、住所検索
- 予算：交通・宿泊・朝食・お土産・追加項目
- 立替精算：メンバー、支払った人、均等割り、送金先の算出
- 準備：持ち物の追加・完了・削除
- 共有：グループ作成・参加・切替、共有メモ、自動同期
- PWA：ホーム画面への追加、オフライン時の基本表示

## 画面の役割と流れ

旅行中に最初に開くホームを入口にして、確認から編集へ自然に進める構成にしています。

1. ホーム：日付、次の予定、移動、宿、参加人数、1人あたりの目安を確認
2. 予定：日ごとの予定を編集し、移動経路とSTAYの住所を地図で開く
3. お金：予算の調整、追加項目、お土産、誰が払ったか、精算結果を管理
4. 持ち物：全員で使うチェックリストを追加・完了・削除
5. 共有：メンバー登録、参加グループの切替、共有メモを管理
6. 設定：旅行名、日程、交通、宿、PWA追加を管理

人数は共有ページのメンバーを唯一の基準にし、ホームの人数表示、旅費合計、1人あたりの予算、立替精算へ連動させます。計算ロジックは `src/derived.ts` に集約しています。

## 次に必要な機能

### 優先度 A：安心して共有するための基盤

1. 同時編集の競合表示：誰かの更新で上書きしそうな時に選べるようにする
2. メンバーごとの本人設定：自分の支払い・持ち物を迷わず入力できるようにする
3. バックアップ：旅行データをJSONで書き出し・読み込みできるようにする
4. 共有権限：閲覧のみと編集可能を分ける
5. オフライン更新キュー：圏外で編集した内容を復帰後に送信する

### 優先度 B：旅行前と旅行中の使いやすさ

1. 予約情報：予約番号、確認URL、PDF・画像を予定に添付する
2. 通知：出発、チェックイン、予約時刻の前に知らせる
3. 行き先候補：地図検索した場所を候補リストへ保存し、投票する
4. 天気と運行情報：旅行日と場所に合わせて確認する
5. 緊急情報：保険、連絡先、病院、紛失時のメモをまとめる

### 優先度 C：旅行後まで楽しむ

1. 写真アルバム：Cloudflare R2へ画像を保存し、日付・場所ごとに並べる
2. 旅の記録：予定を「行った場所」へ変換し、感想を残す
3. 旅行テンプレート：過去の旅行を複製して別の旅行を作る
4. 複数旅行ホーム：今後・進行中・過去の旅行を一覧にする

この順番は「当日すぐ見る情報」から「必要な時だけ編集する情報」へ進む構成です。モバイルの下部ナビは5項目に限定し、設定は右上へ分離しています。

## 追加候補の洗い出し

### 優先度A：旅行前に必要

- 予約情報：予約番号、確認URL、画像・PDF添付
- JSONバックアップ：端末変更や旅行の複製に使える書き出し・読み込み
- 権限管理：閲覧のみ、編集可能の切り替え
- オフライン更新キュー：圏外で編集した内容を復帰後に送信

### 優先度B：旅行中に便利

- 行き先候補：地図検索した場所の保存、投票、予定への追加
- 天気・運行情報：日付と場所に合わせた確認
- リマインダー：出発、チェックイン、予約時刻の通知
- 緊急情報：保険、連絡先、病院、紛失時のメモ

### 優先度C：旅行後に使う

- 写真アルバム：日付・場所・メモ付きの共有アルバム
- 旅の記録：行った場所、感想、写真の保存
- 旅行テンプレート：過去の旅行から新しい旅を作成
- 複数旅行ホーム：予定中・進行中・過去の旅行を一覧表示
